# The-scraper

Парсеры для поска людей по банкротству с сайтов: https://kad.arbitr.ru и https://fedresurs.ru

**Примечание**

Реализован только функционал по сбору данных с https://fedresurs.ru

## Требования

- **Python**: 3.12+
- **inn.csv**: собязательными столбцами url, inn_number, case_number
- **PostgreSQL или SQLite** — для хранения данных из запроса, можно переключать
- **Docker** (опционально) — в проекте есть `Dockerfile` и `docker-compose`

### Установка и запуск (локально)

```bash
cd NAME_PROJECT # в той директории, где у вас хранится проект
pip install -r requirements.txt

python -m app
```

При первом запуске:

- инициализируется соединение с PostgreSQL;
- выполняется план запрсов;

### Запуск в Docker

```bash
# настроить переменные окружения в .env для гибкого управления
docker compose up -d - соборка кластера
```

#### Терминология

Playwright и Selenium — инструменты для автоматизации браузеров, но Playwright быстрее, современнее и стабильнее благодаря прямому управлению через протоколы (Chromium DevTools), обеспечивая автоматическое ожидание элементов. Selenium надежен, имеет огромную экосистему и лучше подходит для поддержки старых браузеров. Playwright эффективнее для современных SPA-приложений.

Ключевые отличия:
Скорость и стабильность: Playwright работает на 35-45% быстрее и реже выдает ошибки "flaky tests" (случайные сбои) за счет встроенных механизмов ожидания элементов.
Архитектура: Selenium использует WebDriver для каждого браузера, а Playwright напрямую работает с браузерным движком, что позволяет быстрее реагировать на события.
Поддержка браузеров: Playwright поддерживает Chromium, WebKit (Safari) и Firefox в одном пакете, в то время как Selenium требует настройки отдельных драйверов.
Работа с элементами: Playwright лучше справляется с Shadow DOM и сложными динамическими элементами.

Автоматические ожидания: В Playwright не нужно явно прописывать ожидания (waits) для большинства действий, в отличие от Selenium.
